Что и как делать:

1. Работа с Git:
    main НЕ ТРОГАТЬ
    Делать ветку от dev (feature-что-хотите-добавить) и делать всё в ней
    Сделали в своей ветке - коммит в GitHub (https://github.com/Diamond-Arrow-the-Lizard/battle-of-hyperborea) и делайте pull request

2. Как с классами:
    Разделять всё по папкам внутри Data (т.е. всё связанное с существами в папке Entiy, способности в Abilities и т.д.)
    Все интерфейсы писать по принципу I{что это}



По игре:

    GUI: сейчас разбираюсь с Avalonia UI, если не получится - погляжу Unity. Советую тоже Avalonia UI посмотреть.
    
    В общих словах: 
        2D.
        Игрок против игрока. 
        Один за Русов, другой - за Ящеров. 
        Сначала всеми юнитами ходит один игрок, потом - второй. 
        Условие победы - все юниты противника побеждены.
        Что-то вроде старых Fire Emblem, если прямо.

    Поле:
        По клеткам
        Может быть пустым, с юнитом или с препятствием
        Имеет координаты
    
    Юниты (существа):
        Координаты

        Здоровье

        Выносливость:
            Зависит от здоровья (75% здоровья == 75% макс выносливости)
            Восстанавливается, если ничего не делать персонажем
            Если падает до нуля - юнит не может атаковать и получает повышенный урон

        Атака:
            Тип атаки:
                Ближний бой - сначала бьёт по выносливости, дальность - одна клетка
                Дальний бой - сразу по здоровью, дальность - до первого препятствия (включая др. юнита)
            Урон
            Бить по диагонали нельзя, только север-юг-восток-запад

        Активные умения: 
            Тип атаки
            Магия или нет: при магической атаки юнит теряет здоровье для применения способности
            Радиус

        Пассивные умения
            Тут хз ещё
